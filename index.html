Abstract

Sparse rewards remain a major challenge in reinforcement learning because agents receive limited feedback during training, making exploration difficult. In this project, we build a sparse reward benchmark to evaluate how different exploration methods affect learning performance. We compare a PPO baseline with intrinsic motivation methods, including Intrinsic Curiosity Module (ICM) and Random Network Distillation (RND), across classic control and robotic manipulation tasks. The benchmark analyzes success rate, average return, learning curves, and state visitation patterns. Our results show that intrinsic rewards can improve exploration coverage, but better exploration does not always lead to faster task success, highlighting the trade-off between exploration diversity and policy learning efficiency under sparse reward settings.

Overview

Sparse-reward environments remain one of the most challenging settings in reinforcement learning because agents receive meaningful feedback only after completing long sequences of correct actions. In such scenarios, successful learning depends not only on efficient exploration, but also on whether the reward structure provides enough information for policy improvement. Although many exploration strategies have been proposed, including entropy regularization, intrinsic curiosity, and random network distillation, their effectiveness often varies substantially across tasks and reward formulations.

A major limitation of existing studies is the lack of a unified evaluation framework. Exploration methods are often tested on different environments, reward designs, and evaluation protocols, making fair comparison difficult. Moreover, commonly used benchmarks such as Atari and classic control tasks are not specifically constructed to examine sparse-reward exploration across tasks with different state and action structures.

To address this issue, we develop a unified sparse-reward benchmark framework for reinforcement learning exploration. The benchmark covers both discrete and continuous-control environments, including Gridworld, FrozenLake, Sparse CartPole, and Meta-World robotic tasks such as Reach and Button. Within a shared training and evaluation pipeline, we compare PPO, PPO+ICM, and PPO+RND using learning curves, success rates, and state visitation patterns. Rather than focusing only on which exploration method performs best, our benchmark is designed to study how exploration mechanisms interact with sparse-reward structures across different task types.

Benchmark Tasks

To systematically evaluate exploration under sparse-reward settings, we construct a benchmark consisting of multiple environments with different task characteristics. Rather than relying on a single reward formulation, the benchmark includes strict sparse rewards, milestone-based rewards, and weak dense shaping in selected tasks, allowing us to study how exploration methods behave under different forms of task guidance.

Gridworld is a deterministic navigation environment in which the agent must discover a distant goal state without intermediate rewards. This task provides a simple setting for visualizing state coverage and exploration behavior.

FrozenLake introduces stochastic transitions, making exploration more challenging and allowing us to evaluate the robustness of exploration methods under environmental randomness.

Sparse CartPole extends the classic CartPole control problem by replacing dense rewards with sparse success signals. This environment combines sparse feedback with continuous-state dynamics.

Meta-World Reach-v3 and Meta-World Button-Press-v3 are robotic manipulation tasks used to evaluate exploration in high-dimensional continuous-control settings. These tasks are studied under sparse or semi-sparse reward formulations, including strict sparse rewards, milestone-based rewards, and weak dense shaping in selected settings. This design allows us to examine how different exploration methods behave when task feedback is limited or only weakly informative.

Together, these environments provide a diverse evaluation suite spanning discrete navigation, stochastic exploration, classic control, and robotic manipulation.

Methods

We use Proximal Policy Optimization (PPO) as the base reinforcement learning algorithm across all benchmark environments. To study exploration under sparse rewards, we compare an entropy-regularized PPO baseline with two intrinsic motivation methods: Intrinsic Curiosity Module (ICM) and Random Network Distillation (RND).

Entropy-Regularized PPO Baseline. PPO learns a policy by optimizing a clipped surrogate objective, which improves training stability by limiting large policy updates. In our benchmark, the PPO baseline also includes entropy regularization, which encourages stochastic action selection and provides a simple implicit exploration mechanism. This baseline serves as the reference method for evaluating how well a standard policy-gradient algorithm can learn under sparse-reward settings without additional intrinsic reward modules.

Entropy-Regularized PPO + ICM. The Intrinsic Curiosity Module provides additional intrinsic rewards based on prediction errors in a learned dynamics model. The agent is encouraged to visit states where its forward model cannot accurately predict the next feature representation. In our benchmark, ICM is added on top of the same entropy-regularized PPO training framework, allowing us to evaluate whether curiosity-driven intrinsic rewards improve exploration beyond the baseline stochastic policy.

Entropy-Regularized PPO + RND. Random Network Distillation generates intrinsic rewards from the prediction error between a fixed random target network and a trainable predictor network. States that are novel to the agent produce larger prediction errors and therefore higher exploration bonuses. As with ICM, RND is combined with the same entropy-regularized PPO backbone so that differences in performance can be attributed more directly to the intrinsic exploration mechanism.

All methods are trained using the same PPO training interface and evaluated under consistent environment settings, training timesteps, and metrics. This allows us to compare learning performance and exploration behavior across methods in a fair and reproducible way.

Experimental Setup

All experiments are conducted under sparse-reward settings or semi-sparse reward setting to evaluate how different exploration strategies affect learning performance. We compare PPO baseline, PPO+ICM, and PPO+RND across multiple benchmark environments, including discrete navigation tasks, classic control tasks, and robotic manipulation tasks.

For discrete and low-dimensional tasks, we adopt pure sparse rewards, where the agent receives external reward only after reaching the goal or completing the task. This setting directly tests whether the agent can discover successful trajectories with minimal feedback.

For high-dimensional continuous-control tasks such as Meta-World Reach-v3, a fully binary sparse reward often makes learning highly unstable because successful states are rarely encountered during early training. Therefore, we investigate multiple sparse or semi-sparse reward designs for Reach-v3, including milestone-based rewards and a weak dense shaping term of 0.005. These settings allow us to examine how different forms of limited task guidance influence the learning behavior of PPO, ICM, and RND.

To ensure fair comparison, all methods are trained using the same PPO training pipeline, evaluation protocol, and environment configuration whenever possible. Training progress is monitored using success rate, average return, and exponentially weighted moving average (EWMA) success rate.

In addition to learning curves, we analyze state visitation patterns using heatmaps. These visualizations help reveal whether an exploration method improves state-space coverage and whether broader exploration leads to better task performance.

Results

Sparse Gridworld

Gridworld is a simple, deterministic navigation environment where the agent must bypass obstacles to reach a goal. While all three methods (PPO, PPO+ICM, and PPO+RND) eventually find the optimal path, their learning speeds and exploration behaviors differ significantly depending on the grid size and the number of obstacles.

20×20 Gridworld with Obstacles

In the 20×20 environment with 25 random obstacles, PPO+ICM is the fastest to reach an EWMA success rate of 0.8. The standard PPO baseline is slightly slower, while PPO+RND struggles the most, taking nearly 1M steps to reach the same level.

When looking at the visitation heatmaps, standard PPO leaves 22 cells in the top-right corner completely unvisited (marked in black). On the other hand, PPO+ICM and PPO+RND explore more broadly, leaving only 9 unvisited cells.

Interestingly, ICM's exploration pattern looks very similar to standard PPO, just with a higher overall exploration rate. RND, however, behaves completely differently: it ignores the bottom-left region early on and only explores the lower and top-right sections much later in the training process.

50×50 Gridworld: Obstacle-Free Exploration

When we scale up to a 50×50 grid without any obstacles, standard PPO actually reaches the 0.8 success rate faster than PPO+ICM. ICM is the first to find the goal (around 206k steps) due to its intrinsic curiosity. However, because of this same curiosity, it keeps exploring the rest of the empty map instead of focusing on the goal. As a result, PPO overtakes it and stabilizes first. Meanwhile, PPO+RND has a success rate of 0 until almost 1M steps.

The Visitation Count heatmaps show that ICM visits the goal area quite often (the yellow area near the target). Standard PPO leaves larger parts of the map unexplored.

RND shows a very strange behavior here: it spends almost all of its time stuck in the corners. This happens because when the agent bumps into a wall and stays in the same place, RND's predictor network still treats it as a prediction error. This gives the agent a "fake curiosity" reward, trapping it in the corners because it's an easy way to get intrinsic rewards without actually exploring.

The First Visit Step heatmaps show that PPO and ICM share a similar exploration order. RND, however, has a dark blue path leading directly to the goal. This proves that RND actually found the goal very early in training. But because the external sparse reward wasn't strong enough to beat the intrinsic curiosity reward, the agent lost interest in the goal once it was visited. It chose to go back to the corners to bump into walls for easy rewards, which explains why it didn't learn the actual task until 1M steps later.

50×50 Gridworld with High-Density Obstacles

When we add 100 obstacles, the task becomes much harder. Standard PPO is still the fastest to succeed, while RND remains the slowest. However, the obstacles force PPO and ICM to take significantly more steps to learn. Since RND was already very slow in the empty map, the obstacles didn't add as much extra training time for it, bringing the final completion times of all three methods somewhat closer together.

While the Visitation Count heatmaps look somewhat similar to the empty map, the First Visit Step heatmaps highlight a big difference in how fast they explore.

Because the agents have to avoid walls, standard PPO takes over 600k steps to explore most of the map, and still misses some spots. In contrast, ICM manages to explore the entire map in just 300k to 350k steps. This shows that ICM's curiosity is very good at helping the agent navigate around complex dead-ends.

RND again shows a different pattern. It finds the goal and maps out some sub-paths very early on. However, just like in the empty map, it struggles to turn this early discovery into a stable policy. It still takes over 1M steps for RND to consistently succeed.

Summary of Gridworld Findings

The Gridworld experiments highlight a clear trade-off: exploring more doesn't always mean learning faster. Intrinsic motivation methods like ICM are great at helping the agent discover the goal quickly in smaller maps. However, in larger environments, too much curiosity can be a distraction. ICM spends too much time exploring empty spaces instead of just going to the goal, and RND can easily fall into the trap of bumping into boundaries for easy rewards. This shows that while intrinsic rewards increase exploration coverage, the agent still needs to balance its curiosity with actually solving the task.

FrozenLake

In the FrozenLake task, PPO with entropy regularization already achieves strong performance under the sparse-reward setting. However, the learning curves show that intrinsic motivation mainly affects the learning speed rather than the final performance.

ICM shows the fastest early improvement in EWMA success rate, suggesting that curiosity-driven rewards help the agent discover successful trajectories earlier in training. PPO also reaches a comparable performance level, but its improvement appears later than ICM. In contrast, RND shows the slowest improvement and performs slightly weaker during the early and middle stages of training.

The visitation heatmaps show that all methods eventually concentrate their visits along the upper row and right column, which correspond to the dominant successful path toward the goal. PPO and RND exhibit relatively similar final visitation patterns, indicating that both methods mainly exploit the successful route after learning.

In contrast, ICM shows noticeably higher visitation frequencies around the starting region and nearby states, especially along the upper-left part of the map. This suggests that ICM does not necessarily increase global exploration coverage, but it improves local exploration around early states, which may help the agent discover successful trajectories earlier in training.

Overall, the FrozenLake results suggest that intrinsic motivation does not significantly improve the final success rate when PPO with entropy regularization already performs well. Instead, ICM mainly improves early learning efficiency, while RND behaves more similarly to PPO and learns more slowly. This highlights the difference between exploration behavior, learning speed, and final task performance in sparse-reward environments.

Sparse CartPole

Sparse CartPole modifies the classic CartPole-v1 task into a sparse-reward exploration problem. The original dense per-step reward is removed; instead, the agent receives a reward of +10 only on the first step at which the cart's x-position crosses a goal threshold of x = 1.5 while the pole remains upright. All other steps yield zero reward. A policy that simply balances the pole at x ≈ 0 — the optimal solution to the original dense task — never receives any reward in our setting, so the agent must learn to simultaneously maintain balance and actively drive the cart toward the goal region.

The learning curves show a clear separation between methods. PPO+ICM and PPO+RND both reach an EWMA success rate of 0.8 at approximately 50k steps, while the standard PPO baseline requires roughly 175k steps to reach the same threshold. This represents a 3-4× sample-efficiency improvement from adding intrinsic motivation. The first successful evaluation episode occurs at around 20k–25k steps for the intrinsic-reward methods, versus 140k–145k steps for the baseline.

The average-coverage curves show that PPO+ICM and PPO+RND quickly expand the agent's visitation across the (cart-position, pole-angle) space, reaching roughly 25–28 unique bins per evaluation episode by 30k steps. The PPO baseline remains around 4 bins for the first 140k steps before expanding to a similar final coverage. This confirms that the intrinsic rewards drive earlier and broader exploration of the state space, which translates into earlier discovery of the goal region.

The state visitation heatmaps reveal where each agent spent most of its time at the end of training. The x-axis shows cart position, the y-axis shows pole angle, and the dashed line marks the goal at x = 1.5. All three methods concentrate visits near x ≈ 0 with the pole close to upright, which reflects the dominant strategy of balancing centrally and making short excursions toward the goal to collect the sparse reward. Because each episode spends most of its steps balancing and only a few steps near the goal, the goal region itself does not appear bright in the heatmaps.

PPO+ICM and PPO+RND show slightly broader horizontal smearing across cart positions and a wider spread across pole angles than the PPO baseline, indicating that the intrinsic-reward methods explored more of the state space during training even though their converged policies eventually settle into the same balance-and-reach behavior.

Summary of Sparse CartPole Findings

On Sparse CartPole, intrinsic motivation provides a substantial sample-efficiency advantage. PPO+ICM and PPO+RND perform almost identically and reach the 0.8 EWMA success threshold roughly 3-4× faster than the PPO baseline. All three methods eventually converge to a success rate of 1.0 with similar final coverage, so the benefit of intrinsic rewards in this task lies entirely in faster discovery of the goal rather than in better final performance. This pattern differs from FrozenLake, where intrinsic motivation mainly affected early learning without changing final outcomes, and from the larger Gridworld settings, where RND struggled with corner-trapping. In a continuous-state task with a clear spatial exploration target, both ICM and RND provide reliable and comparable benefits.

Meta-World Reach-v3

Meta-World Reach-v3 is a robotic manipulation task in which a simulated robot arm must move its end-effector to a target position in three-dimensional space. Because the task involves continuous states, continuous actions, and target-directed motion, it serves as a representative benchmark for sparse-reward exploration in high-dimensional control settings. In our benchmark, Reach-v3 is also used to study how different reward designs, such as milestone-based rewards and weak dense shaping, interact with PPO, ICM, and RND.

Meta-World Reach-v3: Pure Sparse Reward

We first evaluate Meta-World Reach-v3 under a pure sparse-reward setting. In this setting, the original dense reward is removed, and the agent receives a reward only when the target is successfully reached. This makes the task significantly more difficult than original Meta-World setting.

The learning curves show that PPO occasionally discovers successful trajectories, especially in the later stage of training, leading to several spikes in success rate and average return. However, the performance remains unstable. RND shows a short early improvement but fails to maintain consistent success, while ICM shows almost no successful learning under this setting.

The exploration heatmaps further suggest that PPO visits regions closer to the target more frequently, which explains its occasional success. In contrast, ICM and RND tend to concentrate visits around specific workspace regions or boundaries rather than consistently exploring the target area. This indicates that intrinsic rewards do not necessarily produce task-relevant exploration in high-dimensional continuous-control environments.

Overall, the Reach-v3 results show that pure sparse rewards make robotic manipulation substantially harder than discrete navigation tasks. Unlike FrozenLake, where all methods eventually learn reasonable policies, Reach-v3 requires more informative exploration guidance. This motivates the use of controlled sparse rewards or weak shaping terms in later experiments to stabilize training while still preserving the sparse-reward challenge.

Meta-World Reach-v3: Basic Wrapper with Weak Dense Shaping

In this Reach-v3 experiment, we introduce a weak dense shaping term with dense_coef = 0.005 to the pure sparse reward wrapper. Compared with the pure sparse reward setting, this way provides a smoother and more direct learning signal, while still remaining much weaker than a fully dense reward formulation.

The learning curves show that the PPO baseline performs best under this setting. PPO reaches high success rates more quickly and achieves the most stable improvement in both success rate and EWMA success rate. ICM also learns the task to some extent, but its improvement is slower and less stable than PPO. In contrast, RND performs the worst: although it occasionally reaches moderate success rates, it fails to maintain consistent progress and does not converge as reliably as the other two methods.

The exploration heatmaps further support this observation. PPO shows a concentrated visitation pattern around the target region across the x-y, x-z, and y-z projections, indicating that the weak dense shaping term is already sufficient to guide the policy toward effective reaching behavior. ICM also explores around the target region, but its visitation pattern is more diffuse, suggesting that curiosity-driven exploration does not translate into equally efficient exploitation. RND appears visually concentrated in some projections, but this does not lead to strong learning performance.

In particular, PPO exhibits the clearest alignment between target-centered exploration and final task success. ICM reaches similar task-relevant regions, but its visitation remains broader and less stable. RND, despite sometimes showing seemingly focused exploration heatmaps, fails to convert this exploration pattern into sustained reaching success. This suggests that a visually concentrated exploration map does not necessarily imply faster or better policy learning.

Overall, these results indicate that, in the Reach-v3 task, a simple and smooth reward structure with a very small amount of dense guidance is already sufficient for PPO to learn effectively. Under this semi-sparse setting, the additional intrinsic rewards from ICM and RND do not provide clear advantages, and may even interfere with efficient exploitation. Therefore, for relatively simple reaching tasks, reward design appears to have a stronger influence on performance than the choice of intrinsic exploration module.

Meta-World Reach-v3: Milestone Reward without Dense Shaping

In the next Reach-v3 experiment, we introduce a milestone-based reward wrapper that provides additional rewards when the end-effector moves closer to the target. However, the dense shaping reward is still set to zero. This setting provides slightly more guidance than the pure sparse-reward version, but the agent still does not receive continuous distance-based feedback.

The learning curves show that all methods remain unstable under this setting. PPO and RND achieve short periods of success in the early stage of training, but their success rates quickly drop and do not recover consistently. ICM shows a delayed improvement around the middle stage of training, but the improvement is temporary and does not lead to stable task completion.

The exploration heatmaps indicate that the agents still concentrate their visits around limited regions of the workspace. Compared with the pure sparse-reward setting, the milestone reward does not clearly shift the visitation distribution toward the target. Instead, PPO, ICM, and RND all show strong concentration near specific boundaries or corners, suggesting that milestone rewards alone are not sufficient to guide stable target-oriented exploration.

ICM shows particularly strong concentration near the lower-left region, which suggests that curiosity-driven exploration may become trapped in locally novel but task-irrelevant regions. Similarly, PPO and RND do not show a clearly target-centered visitation pattern. This explains why the learning curves only show short success spikes rather than stable improvement.

Compared with the pure sparse-reward setting, milestone rewards do not provide improvement and are still not sufficient to produce stable learning. The results suggest that reaching tasks in high-dimensional continuous-control environments require more informative guidance than sparse success signals or discrete milestone rewards alone. Without dense shaping, the agent may discover the target occasionally, but it struggles to maintain a stable policy.

Meta-World Reach-v3: Milestone Reward with Dense Shaping (0.005)

In the final Reach-v3 experiment, a small dense shaping reward of 0.005 is added on top of the milestone-based reward. Although the shaping reward is extremely small compared with the success reward, it significantly changes the learning dynamics of all methods.

Compared with the pure sparse-reward and milestone-only settings, all methods achieve substantially higher success rates. PPO reaches an EWMA success rate above 0.8 earlier than the intrinsic-motivation methods, indicating that the additional dense reward provides sufficient guidance for efficient policy optimization. ICM also achieves strong performance but converges more slowly, while RND requires the longest training time and does not consistently reach the same success level before training terminates.

The exploration heatmaps reveal distinct behaviors among the three methods. PPO shows the most concentrated visitation pattern around the target region, suggesting that it quickly exploits successful trajectories once the weak dense reward provides useful guidance.

RND exhibits the higher visitation frequency around the target neighborhood, but this does not translate into the best success rate. This suggests that reaching or passing through the target region is not sufficient for successful reaching; the agent must also learn a stable control policy that consistently satisfies the success condition.

ICM appears to visit the target neighborhood less frequently than RND, but it achieves better learning performance. This indicates that ICM may convert exploration into more stable reaching behavior, while RND may continue novelty-seeking around the target and delay exploitation. Overall, these results highlight that target-region coverage and final task success are related but not equivalent.

Meta-World Reach-v3: A result comparison of both basic and milestone reward wrappers with dense shaping

Comparing all reward settings reveals that a small dense shaping signal plays a critical role in high-dimensional sparse-reward robotic manipulation tasks. Under both the milestone-based and basic-wrapper settings, adding a weak dense reward substantially improves learnability compared with the fully sparse case. However, the results further show that reward structure and exploration mechanism interact strongly. With the basic wrapper and dense_coef = 0.005, the PPO baseline learns the Reach-v3 task most effectively, indicating that a simple and smooth directional reward is already sufficient for stable policy learning. In contrast, when intrinsic exploration modules are introduced, both ICM and RND perform better under the milestone-based wrapper than under the basic wrapper, suggesting that these methods benefit more from structured intermediate progress signals. Overall, the findings indicate that performance is determined not only by whether dense shaping is present, but also by how the reward structure matches the underlying learning mechanism.

For the PPO baseline, the basic wrapper produces faster and more stable learning than the milestone wrapper. This suggests that, without intrinsic exploration, PPO benefits most from a simple and smooth reward landscape that directly provides directional guidance.

After adding ICM, the trend reverses. The milestone wrapper substantially outperforms the basic wrapper, indicating that curiosity-driven exploration benefits more from structured intermediate progress signals than from a purely smooth reward design.

A similar pattern is observed for RND. The milestone wrapper again performs better than the basic wrapper, suggesting that novelty-driven exploration also requires more explicit task structure in order to convert exploration into stable task success.

Meta-World: Button-Press-v3

In the Meta-World Button-Press-v3 environment, introducing intrinsic motivation (PPO+ICM and PPO+RND) leads to drastically different behavioral patterns compared to the standard PPO baseline.

Looking at the evaluation learning curves, standard PPO struggles significantly in the early stages. It only begins to successfully press the button around 2.6M steps and takes until approximately 3.5M steps to reach an EWMA success rate of 0.8. In stark contrast, adding intrinsic curiosity accelerates learning immensely: PPO+ICM reaches the 0.8 threshold in just 500k steps, and PPO+RND reaches it at 1.8M steps.

Visualizing 3D Exploration via 2D Projections

To understand why their learning speeds differ so much, we visualize their exploration footprints. Since the robotic arm moves in a 3D space, we project the visitation data onto three 2D planes: XY (top-down view), XZ (front view), and YZ (side view).

For each 2D projection, we aggregate the hidden third axis using two different methods:
1. Min Projection: Represents the "earliest" or "least" interaction. For example, the Min First Visit Step shows the earliest time the agent ever crossed that specific line of sight.
2. Max Projection: Represents the "latest" or "most" interaction. For example, the Max Visitation Count highlights the specific point along that line of sight that was visited the most frequently.

Visitation Count Analysis

The Visitation Count heatmaps reveal clear behavioral habits. Standard PPO tends to fixate and concentrate its movements around a single spatial point. PPO+RND, echoing its problematic behavior in the Gridworld, gets stuck hovering in a specific corner. PPO+ICM, however, is much more dispersed, showing continuous movement and traversal across multiple locations within the workspace.

First Visit Step Analysis

The First Visit Step heatmaps provide further context. In the Min First Visit Step maps, all three methods show blue regions near the button (marked by a star), indicating that they all managed to accidentally wander near the goal early in training.

However, standard PPO and PPO+RND suffer from an "exploration gap." Their maps consist mostly of early (blue) and very late (orange/yellow) colors, missing the smooth transitional colors in the middle. This correlates with our Visitation Count findings: because they get stuck in their preferred single point or corner, they essentially stop exploring new areas for a long period. On the other hand, PPO+ICM continuously seeks out new experiences. Its heatmap shows a smooth, diverse color gradient from blue to yellow, proving it never stopped exploring the workspace.

Summary of Button-Press-v3 Findings

Why does PPO+ICM achieve the fastest breakthrough in this specific task? Unlike simple reaching or navigation tasks, Button-Press-v3 involves physical object interaction. Although all methods initially struggle to find the target, taking a long time to achieve the first successful press, once the robotic arm accidentally presses the button, the environment state changes dynamically (the button goes down). This sudden state change creates a massive prediction error for ICM's forward model, generating a huge intrinsic reward. This "surprise" naturally motivates the agent to keep playing with the button, which explains the sudden vertical spike in its learning curve as it quickly solidifies the successful policy. PPO+RND, however, suffers from its boundary trap vulnerability, getting stuck in corners, while standard PPO simply lacks the active drive to escape its initial movement patterns, delaying their eventual success. This highlights that ICM is particularly powerful and efficient in environments featuring rich, dynamic object interactions.

Conclusion

In this project, we developed a unified sparse-reward benchmark for reinforcement learning exploration across both discrete and continuous-control tasks. Rather than focusing on a single environment or a single exploration strategy, our benchmark provides a reproducible framework for systematically comparing entropy-regularized PPO, PPO+ICM, and PPO+RND under consistent training and evaluation protocols.

Our results show that no single exploration method consistently performs best across all sparse-reward settings. Instead, performance is highly task-dependent and strongly influenced by reward structure. In discrete tasks, exploration mechanisms may improve learning speed without substantially changing final performance, while in continuous-control tasks the relative advantage of PPO, ICM, and RND can change significantly depending on how sparse rewards are formulated.

A particularly important finding of this benchmark is that reward design interacts strongly with the exploration mechanism. In Meta-World Reach, for example, a simple basic wrapper with weak dense shaping is already sufficient to make the PPO baseline highly effective, while milestone-based reward structures are more beneficial when combined with intrinsic exploration methods such as ICM and RND. This shows that sparse-reward exploration cannot be understood only as an algorithm-ranking problem; it must also be analyzed through the interaction between task type, reward structure, and learning dynamics.

In addition, our benchmark demonstrates that exploration quality cannot be judged solely by final success rate. State visitation heatmaps reveal that broader or more target-centered exploration does not always translate into faster or more stable task completion. Therefore, fair evaluation of sparse-reward exploration requires not only multiple task domains, but also consistent metrics that capture both learning outcome and exploration behavior.

Overall, this project highlights the value of benchmark-driven analysis for sparse-reward reinforcement learning. Rather than identifying a universally best exploration algorithm, our benchmark helps reveal when and under what reward structures different exploration mechanisms become effective. We hope this benchmark can serve as a reproducible foundation for future studies on exploration under sparse and semi-sparse reward settings.

Appendix A. Implementation Details

A.1 PPO Training Pipeline

The PPO training procedure is implemented through a lightweight utility function named run_ppo. This function is responsible for constructing the PPO model, launching training, and saving the final checkpoint for later evaluation. The output path is first converted into a Path object, and the corresponding directory is created if necessary. A Stable-Baselines3 PPO model is then initialized using the target environment and the hyperparameters specified in the configuration file.

After initialization, the model is trained through the learn function for a predefined number of timesteps. Optional callbacks can be attached to the training loop, allowing modules such as EWMA-based evaluation, ICM updates, or RND updates to be executed periodically during training. Once training is complete, the trained model is saved to the designated checkpoint path. This design provides a unified training interface across different benchmark environments and exploration settings.

A.2 Training Script and Configuration

The training pipeline is implemented in a central script that loads experiment configurations, builds the environment, applies reward wrappers, initializes exploration modules, and launches PPO training. All experiment settings are defined in YAML configuration files, including environment parameters, wrapper options, agent hyperparameters, training length, output paths, and evaluation settings. This configuration-based design improves reproducibility and allows experiments to be modified without changing the main training logic.

To reduce run-to-run variation, the training script explicitly sets the random seed for Python, NumPy, and PyTorch. The same seed is also applied when resetting both the training and evaluation environments. After constructing the base environment, the script optionally applies sparse-reward wrappers according to the configuration. Exploration modules are then initialized if enabled.

Two intrinsic exploration options are supported in the current implementation: Intrinsic Curiosity Module (ICM) and Random Network Distillation (RND). For both methods, the training script initializes the exploration module, its optimizer, and the corresponding callback for periodic updates. For ICM, the script automatically detects whether the action space is discrete or continuous, and sets the action type and action dimension accordingly. This allows the same training pipeline to support both discrete environments such as FrozenLake and continuous-control environments such as Meta-World.

All active callbacks are collected into a callback list, which may include the ICM update callback, the RND update callback, and the EWMA success evaluation callback. If multiple callbacks are enabled, they are merged into a CallbackList so that they can be executed together during PPO training. This modular design keeps exploration-module optimization separate from policy learning while maintaining a unified overall pipeline.

A.3 Exploration Modules

To improve learning under sparse rewards, we incorporate intrinsic exploration mechanisms on top of PPO. Although both methods aim to encourage broader exploration, they provide additional learning signals in different ways.

A.3.1 Intrinsic Curiosity Module (ICM)

ICM encourages exploration through transition prediction error. It first maps observations into a latent feature space and then trains a forward model to predict the next-state feature from the current-state feature and the action. The intrinsic reward is computed from the forward prediction error. Transitions that are harder to predict produce larger intrinsic rewards, encouraging the agent to explore less predictable dynamics. In addition, an inverse dynamics model is trained jointly to predict the action from consecutive state features, which helps improve the learned representation.

A.3.2 Random Network Distillation (RND)

RND measures novelty through feature prediction mismatch. It uses a fixed randomly initialized target network and a trainable predictor network. For a given observation, the intrinsic reward is defined as the prediction error between the two networks. Observations that are less familiar to the predictor produce larger prediction errors and therefore larger intrinsic rewards. As training proceeds and the predictor learns the visited observations, the intrinsic reward gradually decreases. This mechanism naturally encourages the agent to move toward less familiar regions of the state space.

A.3.3 Comparison of Exploration Signals

ICM and RND emphasize novelty in different ways. ICM rewards transitions that are difficult to predict in a learned dynamics model, while RND rewards observations that remain poorly predicted by the predictor network relative to a fixed random target network. Therefore, ICM is more sensitive to transition-level unpredictability, whereas RND mainly captures state novelty through feature prediction mismatch.

A.4 ICM Module Implementation

The Intrinsic Curiosity Module is implemented to support both discrete and continuous action spaces within a unified framework. The module first encodes each observation into a latent feature representation through a two-layer multilayer perceptron. This feature representation is then used by both the inverse and forward dynamics models.

The inverse model takes the concatenation of the current feature and the next-state feature as input and predicts the action that caused the transition. For discrete action spaces, this is formulated as a classification problem using cross-entropy loss. For continuous action spaces, it is formulated as a regression problem using mean squared error.

The forward model predicts the next-state feature from the current feature and the action. For discrete actions, the action is converted into a one-hot vector before concatenation. For continuous actions, the raw action vector is used directly. The forward model is trained with mean squared error between the predicted next feature and the encoded next feature.

The intrinsic reward is computed from the forward prediction error. In practice, the raw ICM intrinsic reward is further processed before being added to the extrinsic reward. For continuous-action environments, it is normalized by a running standard deviation to stabilize scale during training. Optional clipping can also be applied before scaling and reward combination. The total ICM loss is defined as a weighted combination of inverse loss and forward loss, controlled by the coefficient beta.

A.5 ICM Update Callback

The ICM update callback is used to optimize the Intrinsic Curiosity Module alongside PPO during environment interaction. Instead of updating the module at every step, the callback performs periodic updates according to a predefined update frequency. The exploration wrapper stores transitions of the form (s, a, s') in an internal buffer, and these transitions are retrieved whenever the update condition is met.

When triggered, the callback converts the stored transitions into batched tensors for current observations, actions, and next observations. The action tensor type is adjusted automatically according to whether the action space is discrete or continuous. The callback then calls the ICM module to compute the inverse loss, forward loss, and total ICM loss. Afterward, a standard gradient update is performed by clearing gradients, backpropagating the loss, and updating the ICM parameters through the optimizer. This optimization process is kept separate from the PPO optimizer, allowing curiosity-model learning and policy learning to proceed in parallel.

A.6 RND Module Implementation

The Random Network Distillation module is implemented to generate intrinsic reward based on feature-space prediction error. It consists of two neural networks: a fixed target network and a trainable predictor network. Both networks take the observation as input and output a feature vector of the same dimension.

The target network is randomly initialized and then frozen throughout training. The predictor network is trained to approximate the target network output. For a given observation, the intrinsic reward is defined as the squared prediction error between the predictor output and the target output. States that are less familiar to the predictor receive higher intrinsic reward, encouraging exploration toward novel observations.

In the implementation, the raw RND intrinsic reward is normalized by a running standard deviation before being scaled and added to the extrinsic reward. This normalization helps stabilize the novelty signal during training. The RND predictor is optimized using mean squared error loss between the predictor output and the fixed target output.

A.7 RND Update Callback

The RND update callback is responsible for training the predictor network during PPO interaction. During rollout collection, the environment wrapper stores observations encountered by the agent. These observations are later retrieved through pop_observations() whenever the callback update condition is satisfied.

The callback performs updates periodically rather than at every step. When triggered, the stored observations are converted into a tensor batch on the same device as the RND module. The callback then computes the RND loss and updates the predictor network through a standard gradient step. Since the target network remains fixed, only the predictor network is optimized. As the predictor becomes more accurate on frequently visited observations, the intrinsic reward for those states decreases over time.

A.8 Evaluation Pipeline

The evaluation module is implemented as a benchmark-oriented utility for testing trained agents under a consistent protocol. The function run_one_episode() executes one full episode in the target environment. During interaction, it records the received rewards, whether the task has been successfully completed, the visited coverage identifiers, and the episode length.

An episode is considered successful if any visited state during the episode reports success = True in the info dictionary. To evaluate exploration behavior, the evaluator also tracks all unique coverage_id values encountered during the episode and uses their count as the episode-level coverage metric.

The function evaluate_agent repeatedly runs single-episode evaluation for a predefined number of episodes. It aggregates average return, success rate, average coverage count, and the number of evaluated episodes. This modular evaluation design allows all methods to be compared under the same benchmark criteria.
